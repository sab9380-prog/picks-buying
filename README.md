## 프로젝트 개요

Picks 역기획 바잉 어시스턴트는 백화점·이커머스 OPR 바이어가 해외 브랜드를 매입할 때 필요한 **조닝 판정**, **심리가 산출**, **수익구조 시뮬레이션**을 AI 웹 검색과 결합하여 자동화하는 단일 파일 웹 애플리케이션입니다. 바닐라 JavaScript로 구현되며, 서버 없이 브라우저에서 모든 분석을 수행합니다.

## 해결하는 문제

OPR 바이어는 해외 브랜드 매입 시 다음 작업을 수작업으로 수행합니다:

| 작업 | 수동 처리 시 병목 |
|------|-----------------|
| 브랜드 적정 가격 조사 | 국내외 사이트 5~10개를 일일이 확인 |
| 조닝(판매 전략) 결정 | 경험 기반 판단, 일관성 부족 |
| 수익구조 시뮬레이션 | 엑셀 수식 매번 재작성 |
| 보고서 작성 | 동일 형식 반복 작업 |

**이 도구는 위 과정을 AI 웹 검색 + 알고리즘 기반으로 자동화하여, 브랜드 1건 분석을 약 5분 내로 완료합니다.**

## 주요 기능

### 조닝 판정
BPI(Brand Power Index) 3축 — 선호도·인지도·이용도 — 의 등급(S/A/B/C/F)을 점수로 환산하여 합산한 뒤, 골드라벨/가성비/아이템 조닝으로 자동 분류합니다.

### 심리가 산출
조닝별 할인율을 해외 정상가에 적용한 후, 소비자 심리 최적 가격 16개 포인트(₩9,900 ~ ₩459,000)에 스냅하여 체감 최적가를 도출합니다.

### 수익구조 계산
매입수량·매총율·판매율·판관비 4개 변수를 입력하면, 외형매출부터 영업이익까지 전 구간을 자동 계산합니다. 매입율·TCR(매입원가 대비 매출 비율) 등 실무 KPI를 즉시 확인할 수 있습니다.

### AI 브랜드 분석
Anthropic Claude API의 웹 검색 기능으로 브랜드 정보를 실시간 수집하고, 구조화된 JSON으로 BPI·키아이템·가격 소스를 자동 생성합니다.

## 핵심 알고리즘

### 1. 조닝 판정 (`getZoning`)

BPI 3축의 등급을 점수로 변환하여 조닝을 결정합니다.

```
등급 → 점수: S=35, A=25, B=15, C=8, F=0

총점 = 선호도점수 + 인지도점수 + 이용도점수
선호도점수 = GS[preference]

IF 총점 ≥ 60 AND 선호도점수 ≥ 28:
    → 골드라벨 (프리미엄 포지셔닝)
ELIF 총점 ≥ 35 AND 선호도점수 ≥ 15:
    → 가성비 (가격 경쟁력 포지셔닝)
ELSE:
    → 아이템조닝 (카테고리별 고정가 전략)
```

### 2. 심리가 산출 (`calcPP`)

조닝별로 다른 할인율을 적용한 뒤, 소비자 심리 가격 포인트에 스냅합니다.

```
가격포인트 = [9900, 19900, 29900, 39900, 49900, 59900,
             79900, 99900, 129000, 159000, 199000, 259000,
             299000, 329000, 399000, 459000]

IF 골드라벨:
    심리가 = snap(해외정상가 × 0.38)
ELIF 가성비:
    심리가 = snap(snap(해외정상가 × 0.38) × 0.65)
ELIF 아이템:
    심리가 = 카테고리별_고정가격표[category]

snap(price) = 가격포인트 중 |price - point|가 최소인 값
```

### 3. 수익구조 계산 (`calcRevenue`)

매입부터 영업이익까지 전 과정을 시뮬레이션합니다.

```
입력: qty(매입수량), grossMargin(매총율), sellRate(판매율),
      sga(판관비율), unitPrice(심리가), retailPrice(정상가)

판매수량     = qty × sellRate
외형매출     = 판매수량 × unitPrice          (VAT 포함)
매출액       = 외형매출 / 1.1               (VAT 제외)
매출총이익   = 매출액 × grossMargin
매출원가     = 매출액 × (1 - grossMargin)
매입원가     = 매출원가 / sellRate           (미판매 재고 포함)
매입단가     = 매입원가 / qty
매입율       = 매입단가 / retailPrice
TCR          = 매출액 / 매입원가
영업이익율   = grossMargin - sga
```

## 기술적 의사결정

### Why 단일 HTML?
- 배포 복잡도 제거: `index.html` 하나를 정적 호스팅에 올리면 끝
- 번들링·빌드 도구 불필요: 의존성 관리 오버헤드 제거
- 이식성: 파일 하나를 공유하면 누구든 즉시 사용 가능

### Why 바닐라 JS?
- 프레임워크 학습 곡선 없이 도메인 로직에 집중
- 런타임 투명성: 모든 코드가 한 파일에서 추적 가능
- 번들 크기 최소화: React/Vue 없이 ~220KB (CSS+JS+데이터 포함)

### Why 클라이언트사이드 LLM 호출?
- 서버 인프라 비용 제로
- API 키를 사용자가 관리 → 서버에 키 저장 불필요 (보안 단순화)
- 트레이드오프: 사용자가 직접 API 키를 입력해야 하는 UX 마찰

## 사용 방법

### 데모 모드
1. [picks-buying.pages.dev](https://picks-buying.pages.dev) 접속
2. 상단 **"데모 모드 시작"** 클릭
3. 샘플 브랜드 3개가 자동 로드되어 조닝·심리가·수익구조를 즉시 확인 가능
4. 데모 데이터는 사전 저장된 응답이며, 실제 API 호출 없음

### 실제 사용
1. 우측 상단 🔑 버튼 → Anthropic API 키 입력
2. 검색창에 브랜드명 입력 (예: "아크네 스튜디오") → AI 분석 시작
3. 분석 완료 후 브랜드 카드에서 조닝·심리가·수익구조 확인
4. 📊 Excel 버튼으로 역기획표 다운로드

## 한계와 향후 계획

### 알려진 한계
- **API 키 관리:** 사용자가 직접 Anthropic API 키를 입력해야 함
- **BPI 정확도:** AI 추정치이며, 실제 내부 설문/판매 데이터와 차이가 있을 수 있음
- **실측 ROI 없음:** 수익구조는 시뮬레이션이며 실제 매출과 다를 수 있음
- **포트폴리오 규모:** 45개 브랜드 기준 설계·테스트 완료, 100개+ 미검증
- **환율:** 외부 API 기반, 실시간 은행 고시 환율과 차이 있음
- **모바일:** 데스크톱 환경 권장

### 향후 계획
1. **서버사이드 API 프록시** — API 키 관리를 서버로 이전, 보안 강화 + 팀 공유
2. **SAP/ERP 연동** — 실판매 데이터 반영으로 BPI를 추정치에서 실측치로 전환
3. **멀티시즌 포트폴리오** — 시즌별 매입 계획 비교·최적화
